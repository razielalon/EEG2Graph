Analysis of preprocess_zuco.py and Dataset Structure

1. Script Overview:
   The script `preprocess_zuco.py` is designed to process ZuCo (Zurich Cognitive Language Processing Corpus) data from MATLAB (.mat) files into a format suitable for graph-based machine learning models.
   
   It performs the following steps:
   - Iterates through `*_TSR.mat` files in a directory.
   - Loads sentence data and relation labels.
   - Extracts subject and object entities using spaCy (heuristic: first entity = Subject, last = Object).
   - Extracts EEG features for each word, specifically looking for `mean_g1` (Gamma band mean).
   - Saves the processed dataset as a pandas DataFrame in a pickle file (`zuco_graph_dataset.pkl`).

2. Dataset Structure Analysis:
   - Path: The user pointed to `../dataset`. The relevant files were found in `../dataset/Matlab files/`.
   - File format: The files are named `resultsZ*_TSR.mat` (e.g., `resultsZAB_TSR.mat`).
   - Internal Structure: 
     - The files contain a `sentenceData` struct.
     - Each item has `content` (text), `word` (array of word structs).
     - The `word` structs in the provided dataset DO NOT contain the `mean_g1` field that the script expects.
     - Available fields in `word` include eye-tracking specific features combined with band names, such as:
       `FFD_a1`, `FFD_a2`, `FFD_b1`, `FFD_b2`, `FFD_g1`, `FFD_g2`, `FFD_t1`, `FFD_t2`
       (FFD = First Fixation Duration, GD = Gaze Duration, GPT = Go-Past Time, SFD = Single Fixation Duration, TRT = Total Reading Time).
     - There is also a `rawEEG` field.

3. Identified Mistakes & Issues:

   A. Wrong Dataset Path:
      - Script: `processor = ZuCoProcessor(data_dir=".")` (Line 168)
      - Issue: The script looks in the current directory, but the data is in `../dataset/Matlab files`.
      - Fix: Update `data_dir` to point to the correct absolute or relative path.

   B. Missing `mean_g1` Feature:
      - Script: `features = word.mean_g1` (Line 75)
      - Issue: The field `mean_g1` does not exist in the inspected .mat files. The script will currently fill the dataset with zero-vectors only (Line 92).
      - Fix: Determine which feature corresponds to the intended "Gamma band" signal. Possible candidates are `FFD_g1` (Gamma power during first fixation) or `TRT_g1` (Gamma power during total reading time), or computing the mean from `rawEEG`.

   C. Heuristic Entity Extraction:
      - Script: `ents[0], ents[-1]` (Line 55)
      - Issue: Assuming the first entity is the Subject and the last is the Object is a weak heuristic and will likely produce incorrect labels for complex sentences.
      - Recommendation: Use the `relation` field or dependency parsing for more accurate extraction if ground truth is available.

   D. Memory Usage:
      - Script: Appends all processed data to `self.dataset` list.
      - Issue: For very large datasets, this might consume too much RAM. Since ZuCo is relatively small, this is likely acceptable, but worth noting.

4. Recommendations:
   - Modify the script to point to `../dataset/Matlab files/`.
   - Update the feature extraction logic to use available fields (e.g., `TRT_g1` or `FFD_g1`) or derive features from `rawEEG`.
   - Add error checking to warn if features are all zeros.
